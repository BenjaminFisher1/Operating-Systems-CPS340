2. *How are the results different when using a WORST fit policy to search the free list (-p WORST)? What changes?*
Worst fit will always use the largest block available, and the fragmentation is in larger blocks

3. *What about when using FIRST fit (-p FIRST)? What speeds up when you use first fit*
First fit simply chooses the first block large enough to fit the request.
Lookup time speeds up because it no longer has to check the entire free list to determine fit.

4. *For the above questions, how the list is kept ordered can affect the time it takes to find a free location for some of the policies. Use the different free list orderings (-l ADDRSORT, -l SIZESORT+, -l SIZESORT-) to see how the policies and the list orderings interact*
 ADDRSORT sorts by mem address, SIZESORT+ sorts blocks in ascending order, SIZESORT- sorts blocks in descending order.
SIZESORT- is like worst fit, SIZESORT+ is best fit, and ADDRSORT is sort of similar to first fit.
